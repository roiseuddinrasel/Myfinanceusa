# 💰 MyFinance — Personal Accounting App

[![React](https://img.shields.io/badge/React-18-61DAFB?logo=react&logoColor=white)](https://react.dev)
[![Vite](https://img.shields.io/badge/Vite-5-646CFF?logo=vite&logoColor=white)](https://vitejs.dev)
[![Recharts](https://img.shields.io/badge/Recharts-2-22B5BF)](https://recharts.org)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

A clean, modern **personal finance tracker** built with React. Track your income, expenses, and budgets — with interactive charts, budget alerts, and a dark mode — all stored locally in your browser with zero backend required.

---

## ✨ Features

- **Dashboard** — stat cards (balance, income, expenses, savings rate), monthly bar chart, category donut chart, recent transactions list
- **Income Management** — add / edit / delete income entries (source, amount, date, notes)
- **Expense Management** — add / edit / delete expenses across 6 categories; filter by category
- **Reports** — 6-month area trend chart, category pie chart, budget vs actual progress bars
- **Budget Alerts** — live warnings when any category hits ≥ 80 % of its monthly limit
- **Editable Budgets** — set per-category limits directly from the Reports page
- **Dark Mode** — full theme toggle, persisted across sessions
- **Print / Export** — `window.print()` from the Reports page for PDF export
- **Sample Data** — 40+ realistic transactions pre-loaded on first launch
- **LocalStorage Persistence** — all data survives page refresh, no server needed

## 🚀 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) v18 or later
- npm v9 or later (comes with Node.js)

### Installation

```bash
# 1. Clone the repo
git clone https://github.com/YOUR_USERNAME/myfinance.git
cd myfinance

# 2. Install dependencies
npm install

# 3. Start the development server
npm run dev
```

Open **http://localhost:5173** in your browser.

### Demo Login

```
Email:    demo@myfinance.com
Password: demo123
```

Credentials are pre-filled on the login screen — just click **Sign In**.

## 🏗 Build for Production

```bash
npm run build      # outputs to dist/
npm run preview    # preview the production build locally
```

Deploy the `dist/` folder to any static host — [Vercel](https://vercel.com), [Netlify](https://netlify.com), [GitHub Pages](https://pages.github.com), or your own server.

### Deploy to GitHub Pages (one-time setup)

```bash
npm install --save-dev gh-pages
```

Add to `package.json`:
```json
{
  "homepage": "https://YOUR_USERNAME.github.io/myfinance",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
```

Add to `vite.config.js`:
```js
export default defineConfig({
  plugins: [react()],
  base: '/myfinance/',   // ← your repo name
})
```

Then deploy:
```bash
npm run deploy
```

## 🗃 Data Model

```js
// Income entry
{
  id:     "t1234567890_ab1c",   // generated by genId()
  type:   "income",
  amount: 5000,                 // number, USD
  source: "Salary",             // Salary | Freelance | Investment | Business | Gift | Others
  date:   "2026-04-01",         // YYYY-MM-DD string
  notes:  "Monthly salary"      // optional free text
}

// Expense entry
{
  id:       "t9876543210_xy2z",
  type:     "expense",
  amount:   320,
  category: "Food",             // Food | Rent | Transport | Utilities | Education | Others
  date:     "2026-04-05",
  notes:    "Weekly groceries"
}
```

All data is stored under `localStorage` key `"pa_app_v2"` as a JSON object with keys `{ user, transactions, dark, budgets }`.

## 🔧 Customisation

### Add a new expense category

```js
// src/constants.js
export const CATEGORIES = ['Food', 'Rent', 'Transport', 'Utilities', 'Education', 'Health', 'Others']

export const CAT_COLORS = {
  ...
  Health: '#EC4899',
}

export const DEFAULT_BUDGETS = {
  ..., Health: 200,
}
```

### Use the real current date

```js
// src/constants.js  — change this line:
export const TODAY = new Date().toISOString().split('T')[0]
```

### Connect to a real backend

Replace the `useEffect` blocks in `src/hooks/useTransactions.js` with `fetch` / `axios` calls to your Express, Firebase, or Supabase API. The data model is clean and ready to map directly to REST endpoints or Firestore documents.
